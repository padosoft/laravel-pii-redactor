# Regola: Naming Conventions

- Classi: PascalCase
- metodi e funzioni: camelCase
- variabili locali: nomi espliciti, non abbreviazioni arbitrarie
- file: coerenti con il linguaggio e le convenzioni del repo
- directory: descrivono il modulo o il dominio, non il tool

## Regole pratiche

- Il nome deve spiegare il ruolo, non il tipo generico.
- Evita `helper`, `manager`, `data`, `common` se puoi essere piu' preciso.
- Se il repo usa naming diverso, segui il repo invece di imporre questo file.
