# Regola: Pull Request Workflow

- Lavora su branch dedicato.
- Mantieni PR focalizzate su un obiettivo coerente.
- Esegui test proporzionati alla modifica.
- Documenta rischi, breaking changes e migrazioni richieste.
- Non mischiare refactor non necessari con fix puntuali senza dirlo.

## Prima di aprire o aggiornare una PR

- verifica diff finale
- rimuovi debug e file temporanei
- segnala punti non testati o dipendenze esterne
- se restano dubbi architetturali, scrivili nella descrizione
